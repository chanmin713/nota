# Markdown 입출력 규칙(초안)

## 범위(US1)
- 호환 대상: Heading, List(불릿/번호), Code Block, Inline Code, Link, Table(기본)
- 내보내기 우선, 가져오기는 기본 매핑만(고급은 후속)

## 내보내기 규칙
- Heading: H1–H3 → `#`, `##`, `###`
- List: 불릿 → `- `, 번호 → `1. ` (중첩은 2레벨까지 보장)
- Code Block: fenced ``` (언어 태그 보존)
- Inline Code: 백틱(`) 감싸기
- Link: `[text](target)` (내부 IRI는 표시 텍스트만 유지 또는 별칭 → URL 변환 규칙 별도)
- Table: 파이프(`|`) 표, 헤더 구분선 포함

## 붙여넣기 인식(개요)
- 선행 `#`, `- `, `1. ` 자동 인식 → 구조화 변환(충돌 시 사용자 취소 우선)

## 가져오기(기본)
- 위 항목 역매핑, 미지원 요소는 인라인 텍스트로 보존
- 링크 IRI 매핑은 후속(온톨로지/링크 정책과 정합 필요)

## 예외/한계
- 고급 표(병합/정렬), 각주, Mermaid/수식/인용 등은 M2 이후 처리
- 공백/인코딩: UTF-8, 개행 표준화 유지
